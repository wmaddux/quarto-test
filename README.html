Aerospike Health Analyzer (v1.3.0)

The Aerospike Health Analyzer is a diagnostic orchestration tool designed to ingest Aerospike telemetry into a structured SQLite database, execute automated health checks (Rules), and generate professional HTML reports via Quarto.

Application Flow

The application operates as a three-stage pipeline to transform raw logs into actionable insights:

  1. Ingestion: Raw telemetry is processed by the manager. Data is normalized and stored in aerospike_health.db.
  2. Analysis (Rules): The engine executes independent logic modules from the rules/ directory against the database.
  3. Reporting: Quarto processes report.qmd, aggregating rule results, metadata, and visualizations into a self-contained report.

Component Map

Core Logic & Ingestion

Component Role in Flow
ingest_manager.py The Architect. Handles the high-level logic of finding bundles, setting up the SQLite schema, and managing the overall ingestion lifecycle.
run_ingest.py The Orchestrator. The primary CLI entry point. It calls the manager and specific parsers to process the data.
ingest/ The Specialists. Specialized parsers (e.g., node_stats, config_ingest) that translate raw Aerospike text/logs into structured SQL rows.
set_version.py The Synchronizer. Ensures version numbers and dates remain consistent across all scripts and documentation.

Reporting & Analysis

Component Role in Flow
rules/ The Logic Library. Independent modules that query the DB to identify specific health issues (e.g., skews, limits, drifts).
report.qmd The Brain. The Quarto template that executes the rules, processes results, and renders the final UI.

Inside the Report Logic (report.qmd)

The report.qmd file is more than a template; it is the execution engine for the final analysis:

  • Rule Execution: It imports and runs every check in the rules/ directory.
  • Identification & Categorization: It captures the status (PASS, WARNING, CRITICAL) and maps findings to specific cluster impacts.
  • State Summarization: It aggregates high-level metrics (e.g., Total Delete Not Founds, Node Count) for the Executive Summary.
  • Dynamic Remediation: It pulls specific actionable steps from each rule so the user knows exactly how to fix identified issues.
  • Visualization: It uses Plotly to generate interactive charts from the database, allowing for visual identification of performance skews.

Installation & Setup

1. Clone the Repository

git clone https://github.com/your-repo/aerospike-health-analyzer.git
cd aerospike-health-analyzer

2. Prerequisites

  • Python 3.8+
  • Quarto CLI: Download and Install Quarto

3. Install Python Dependencies

pip install pandas plotly

Ingestion & Versioning

Current Sources

  • Collectinfo Bundles: Supports standard Aerospike .tgz or .tar bundles.
  • System Stats: OS-level telemetry (e.g., ENA support, disk utilization).

Future Roadmap (Static File Support)

  • aerospike.conf: Direct ingestion to detect drift against the running state.
  • Log Ingestion: Parsing aerospike.log for migration and heartbeat patterns.
  • System Logs: Ingesting dmesg to correlate hardware events with cluster stability.

Extending and Maintaining

New diagnostics can be added by following the RULE-TEMPLATE.md: 1. Create a new .py file in rules/. 2. Ensure it returns the required schema: id, name, status, message, and remediation. 3. Register the module in the setup block of report.qmd.

Usage (v1.3.0)

# 1. Place bundles in bundles/ directory
# 2. Ingest
python3 run_ingest.py bundles/telemetry_file.tgz

# 3. Generate Report
quarto render report.qmd
